Declarative programming is when you write your code in such a way that it describes what you want to do, and not how you want to do it

Declarative views make your code more predictable and easier to debug.

React components

What component is?

Components let you split the UI into independent, reusable pieces.

Conceptually, components are like JavaScript functions.

They accept inputs that are called "props" and return React elements.

Function and Class Components

Functional component:

              
        function Welcome (props) {
          return 
 Hello, {props.name}
;
        }

Class component

              
          class Welcome extends React.Component {
            render() {
              return 
 Hello, {this.props.name}
;
            }
          }

State

React components has a built-in state object.

The state object is where you store property values that belongs to the component.

When the state object changes, the component re-renders.

Creating the state Object

The state object is initialized in the constructor:

              
          class Car extends React.Component {
            constructor(props) {
              super(props);
              this.state = {brand: "Ford"};
            }
          }

Using the state Object

You can refer to the state object anywhere in the component by using the this.state.propertyname syntax:

              
                

                  
My {this.state.brand}

                  

                    It is a {this.state.color}

Changing the state Object

To change a value in the state object, use the this.setState() method.

When a value in the state object changes, the component will re-render.

              
                changeColor = () => {
                  this.setState({color: "blue"});
                }
